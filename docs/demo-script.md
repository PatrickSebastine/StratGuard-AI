“A backtest can look profitable while quietly relying on ambiguous rules, unrealistic costs, or trade timing that could never have happened. StratGuard AI is a research-only developer tool that makes those weaknesses visible before deployment.”

Open the dashboard. Explain the 20/50 SMA demo, fee and slippage assumptions, and the explicit next-candle execution rule. Run the demo audit.

## 1:00-1:45: Evidence

Show metrics, recorded trades, and credibility findings. State that historical returns do not prove future profitability and that limited trade count or unrealistic assumptions must be visible.

## 1:45-2:25: AI boundary

Explain that GPT-5.6 is intended to return structured strategy specifications and audit the stored evidence. Python produces prices, trades, and metrics deterministically. This boundary prevents the model from inventing financial calculations.

State that Codex accelerated the tested implementation and documentation. Close with the practical outcome: strategy developers can challenge false confidence before paper trading or deployment.
